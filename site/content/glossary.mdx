# Glossary

```txt
resolver -> source -> variable -> value
metadata -> requirements -> audit -> graph
```

### Audit

A static report of executable and external surfaces. See [audit schema](/schemas/audit).

### Config path

The dot-path location of a value in the parsed config object. See [graph schema](/schemas/graph).

### Fallback

An alternate value in a variable expression, such as `${env:HOST, "localhost"}`. See [resolution model](/concepts/resolution-model).

### Requirement

A normalized input contract derived from metadata. See [requirements schema](/schemas/requirements).

### Safe mode

A policy that blocks executable files, custom code, dotenv mutation, and file traversal during resolution. See [security policies](/security-policies).

### Source class

The broad category of a value source: `user`, `config`, `readonly`, or `remote`. See [introspection model](/concepts/introspection-model).
