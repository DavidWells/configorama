# Variable types

Configorama variables use `${...}` syntax to describe where a value should come from. The common shape is `${source:name, fallback}` for keyed sources and `${source(...)}` for function-style helpers.

```yaml
stage: ${opt:stage, "dev"}
host: ${env:API_HOST, "localhost"}
schedule: ${cron(every 5 minutes)}
```

| Type | Source class | Page |
|---|---|---|
| `env` | `user` | [Environment](/variables/env) |
| `opt` / `option` | `user` | [Options](/variables/options) |
| `param` | `user` | [Params](/variables/params) |
| `self` / bare path | `config` | [Self](/variables/self) |
| `file` | `config` | [File](/variables/file) |
| `text` | `config` | [Text](/variables/text) |
| `git` | `readonly` | [Git](/variables/git) |
| `cron` | `readonly` | [Cron](/variables/cron) |
| `eval` | `readonly` data-flow | [Eval](/variables/eval) |
| `if` | `readonly` data-flow | [If](/variables/if) |

Custom variable sources are extension points rather than core types. See [write a custom variable source](/guides/custom-variable-sources) for the user-provided resolver API and [safe inspection](/guides/inspect-config#audit-risk) for execution policy.
