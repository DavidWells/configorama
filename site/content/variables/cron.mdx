# Cron variables

`cron(...)` converts common human-readable schedules into cron expressions. Existing cron expressions pass through as cron strings.

{/* docs CONFIGORAMA_EXAMPLE id="variable-cron" lang="yaml" */}
```yaml
everyFiveMinutes: ${cron(every 5 minutes)}
# ^ becomes */5 * * * *
weekdayMorning: ${cron(on monday at 9:00)}
# ^ becomes 0 9 * * 1
dailyMidnight: ${cron(every day)}
# ^ becomes 0 0 * * *
businessHours: ${cron(business hours)}
# ^ becomes 0 9-17 * * 1-5
monthlyBilling: ${cron(on 1st of month at 00:00)}
# ^ becomes 0 0 1 * *
existingCron: ${cron(15 10 * * 1-5)}
# ^ stays 15 10 * * 1-5
```
{/* /docs */}

Use cron variables when deployment config wants readable schedules but the target platform expects cron syntax. Supported phrases include common intervals, weekdays, named times such as `midnight`, and explicit times.

See [filters and functions](/filters-functions) for related helpers.
