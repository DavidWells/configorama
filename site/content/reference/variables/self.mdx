# Self variables

`self` variables read from the current config object. Bare config paths can also resolve as self references when they match values in the config.

{/* docs CONFIGORAMA_EXAMPLE id="variable-self" lang="yaml" */}
```yaml
service: billing
baseUrl: https://${self:apiHost}:${self:apiPort}
serviceUrl: ${self:baseUrl}/${self:stage}/${self:service}
```
{/* /docs */}

Self references are config-derived values, not external inputs. They can point at values that themselves contain variables, so resolution may take multiple generations.

See [resolution model](/concepts/resolution-model) for generation behavior and [debug resolution](/guides/debug-resolution) for path-focused troubleshooting.
